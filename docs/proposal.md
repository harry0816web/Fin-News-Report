# 台灣財經新聞每日彙整 App — 專案提案

> 版本：v1.0  
> 日期：2026-05-16  
> 狀態：草稿

---

## 1. 專案目標

建立一套自動化系統，每天早上抓取台灣財經相關新聞，透過 **Gemini Flash** 進行 AI 摘要與影響分析，並以 **網頁** 及 **Email 訂閱** 兩種方式呈現給小團隊使用，部署於 **Azure for Education（免費額度）**。

---

## 2. 需求確認摘要

| 項目 | 確認內容 |
|------|---------|
| 新聞來源 | 主要台灣財經媒體（系統自動選定，見第 3 節） |
| 新聞類別 | 科技股市、總體經濟、上市公司公告、國際財經（影響台灣部分） |
| 產業聚焦 | 科技業為主（半導體、電子、科技公司） |
| 執行時間 | 每天早上 09:00（台灣時間，UTC+8） |
| 呈現方式 | 網頁（標題列表 → 點擊展開詳細分析）+ Email 訂閱 |
| 輸出語言 | 繁體中文 |
| 摘要深度 | 新聞摘要 + 對台灣市場的潛在影響分析 |
| 歷史保留 | 最近 7 天 |
| 使用者 | 小團隊（< 10 人），無需登入，Email 自助訂閱 |
| Azure 預算 | 盡量控制在免費額度內，不超出 |

## 3. 新聞來源規劃

以下來源提供 **RSS Feed** 或可爬取，涵蓋各類台灣財經資訊：

| 媒體 | 類型 | 取得方式 |
|------|------|---------|
| 經濟日報 (udn.com) | 總體 / 股市 | RSS |
| 工商時報 (ctee.com.tw) | 總體 / 產業 | RSS |
| 鉅亨網 Cnyes | 股市 / 國際 | RSS |

## 6. Gemini Flash Prompt 設計方向

每批新聞送入 Gemini Flash 時，將附帶以下指令：

1. **篩選**：從所有抓取的新聞中，挑選最重要的 10-15 則
2. **分類**：歸入「科技股市、總體經濟、上市公司公告、國際財經」四類
3. **摘要**：每則新聞產生 2-3 段繁體中文摘要
4. **影響分析**：針對每則重要新聞，分析對台灣股市/科技業的潛在影響（正面 / 負面 / 中性）
5. **輸出格式**：結構化 JSON，供前端直接使用

## 7. 技術選型

| 層級 | 技術 | 理由 |
|------|------|------|
| 語言 | Python（現有 uv 專案） | 已有環境，生態完整 |
| AI | Gemini Flash（gemini-1.5-flash 或 gemini-2.0-flash） | API Key 已就緒，速度快成本低 |
| 爬蟲 | `feedparser` + `httpx` | RSS 解析首選，輕量 |
| 後端 API | Azure Functions（Python） | 免費額度充足，含定時觸發 |
| 前端 | Vue 3 + Vite（靜態） 或 純 HTML/JS | 部署至 Azure Static Web Apps 免費方案 |
| 儲存 | Azure Blob Storage | 存放 JSON、訂閱名單，極低成本 |
| Email | Azure Communication Services Email | 免費 100 封/天，符合小團隊需求 |
| 排程 | Azure Functions Timer Trigger | 內建 cron，無需額外服務 |

## 8. Azure 資源與費用估算

| 資源 | 方案 | 免費額度 | 預估用量 |
|------|------|---------|---------|
| Azure Functions | Consumption | 1M 次執行 / 月，400,000 GB-s | 每天 1 次，遠低於上限 ✅ |
| Azure Blob Storage | LRS | 前 12 個月 5 GB 免費 | 每日 JSON < 1 MB，7 天 < 10 MB ✅ |
| Azure Static Web Apps | Free 方案 | 永久免費 | ✅ |
| Azure Communication Services | Email | 100 封 / 天 | 小團隊訂閱 ✅ |
| **預估月費** | | | **接近 $0** |

> ⚠️ Education 帳號通常有 $100 美元額度，即使有少量超出也有緩衝。
